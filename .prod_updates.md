# Talky 产品与研发进展更新（2026-03-22）

## 1. 重要背景（产品定位与核心逻辑）

- 产品定位：`Talky` 是面向 macOS（Apple Silicon）的语音输入助手。
  - **开源本地版（Free）**：ASR + LLM 在本机运行，用户自行下载模型。
  - **云端订阅版（Cloud）**：音频发送至 Talky Cloud 服务器处理，用户无需本地模型。
- 核心链路：按住热键录音 → 松开触发处理（本地或云端）→ 自动粘贴（无焦点则弹浮窗复制）。
- 核心模块：
  - 控制层：`main.py` + `talky/controller.py`
  - ASR：`talky/asr_service.py`（本地 Whisper）
  - LLM：`talky/llm_service.py`（本地 Ollama）
  - 云端处理：`talky/remote_service.py`（Cloud 模式，POST 音频到服务器）
  - 服务端：`talky-server/main.py`（FastAPI，部署在 Mac Mini）
  - 录音/粘贴：`talky/recorder.py`、`talky/paster.py`
  - 配置/历史：`talky/config_store.py`、`talky/history_store.py`
- 配置持久化：用户设置（含词典、模型、热键、Cloud 配置）保存在 `~/.talky/settings.json`。
- 商业模式：Freemium — 开源免费本地版 + 付费云端订阅（比竞品 Typeless $12/月 更低定价）。

### D. Talky Cloud 服务端（2026-03-21 新增）
- `talky-server/main.py`：FastAPI 服务，部署在 Mac Mini（M1 16GB）。
  - `POST /api/process`：接收音频 + API Key → Whisper ASR → Ollama LLM → 返回清理后文本 JSON。
  - `GET /api/health`：健康检查。
  - `asyncio.Semaphore(1)`：GPU 串行化，避免并发推理冲突。
  - 音频在内存处理，不落盘（隐私保护）。
  - 启动时自动预热 Whisper + Ollama。
- `talky-server/api_keys.json`：用户密钥（已在 `.gitignore`，不入仓库）。
- `talky-server/api_keys.example.json`：密钥模板。
- `talky-server/requirements.txt`：依赖清单。

自动粘贴 ←──(JSON)──── {"text":"..."}

### Mac Mini 服务端启动

**局域网模式（当前）：**
```bash
cd ~/talky && source .venv/bin/activate
export OLLAMA_HOST=http://127.0.0.1:11434
python talky-server/main.py
# 确保 Ollama 在另一终端运行: ollama serve
```

**外网模式（Cloudflare Tunnel）：**
```bash
# 首次：注册 Cloudflare + 购买域名后运行
cd ~/talky && ./talky-server/setup_tunnel.sh api.yourdomain.com

# 日常启动（server + tunnel 一起）：
cd ~/talky && ollama serve &
./talky-server/start_cloud.sh
```

- Python 3.12（Homebrew `/opt/homebrew/opt/python@3.12`）+ venv（`~/talky/.venv`）
- Mac Mini IP: `192.168.31.210`（局域网，可能因 DHCP 变化）
- API Keys: `~/talky/talky-server/api_keys.json`（不在 Git 中）

### 客户端切换模式

- Settings → **Processing Mode** → Local / Cloud
- Cloud 填入 URL（局域网 `http://192.168.31.210:8000` 或外网 `https://api.yourdomain.com`）+ API Key
